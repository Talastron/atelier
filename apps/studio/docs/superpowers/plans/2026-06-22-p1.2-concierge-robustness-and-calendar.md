# P1.2: Concierge Robustness + Calendar & Lookbook Polish

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** Close five user-reported gaps from P1.1 testing — three of them are showstoppers, two are quick polish:

1. **Concierge looks hung during long thinks.** The streaming-placeholder shows three 6px grey dots. For a "pack for a 4-day trip" question that takes 30-60s, those dots are invisible and the user abandons. Need a prominent in-bubble "Composing your reply…" with rotating stage labels (like the Lookbook AIProgressModal) + elapsed-seconds counter after 5s.
2. **Orphan-message state after mid-stream navigation away.** Closing the panel mid-stream cancels the save, leaving a saved user-message with no assistant reply and no recovery. Need a "Tap to retry" affordance when the last message in a thread is a user message with nothing after it (and we're not currently streaming).
3. **Auto-name on AI-composed outfits.** Today the user has to click "Suggest" after the Stylist composes — extra friction. Auto-name during the AI flow; keep the Suggest button for regeneration.
4. **Calendar — can't see what's planned for a day.** Clicking a day must surface the planned outfit prominently, with the packing list one tap away.
5. **Lookbook header stats overflow.** Top-right "Lookbook · 27" / subtitle text overruns its container on smaller widths — needs `min-w-0` + `truncate`.

**Architecture:** Two independent branches off main.
- **TG1** (`p1.2/concierge-robustness`) — touches the Concierge placeholder rendering and the post-thread-load orphan detection. Doesn't touch the stream function itself.
- **TG2** (`p1.2/calendar-and-lookbook-polish`) — three smaller polish changes: auto-name in `handleAIStyle`, calendar selected-day panel beef-up + persistent packing list, lookbook stats CSS truncation.

**Tech Stack:** Same as P1/P1.1 — React 18, Vite 6, Tailwind 4, Firebase 11, lucide-react.

```javascript
// Prominent in-bubble "Composing your reply…" indicator. Renders while the
// streaming placeholder exists but no text has arrived yet (the worst UX
// window — feels like a hang on a slow first chunk). Rotates through
// stage labels every 1.5s + reveals an elapsed-seconds counter after 5s
// so the user knows it's still working, even on a long compose.
//
// Disappears as soon as the first chunk lands (then the streaming text +
// caret take over the bubble).
function ConciergeComposingIndicator() {
  const STAGES = [
    'Reading the conversation…',
    'Looking through your wardrobe…',
    'Considering the moment…',
    'Composing a reply…',
    'Adding finishing touches…',
  ];
  const [stageIdx, setStageIdx] = useState(0);
  const [elapsedMs, setElapsedMs] = useState(0);
  const startRef = useRef(0);

  useEffect(() => {
    startRef.current = performance.now();
    const stageTimer = setInterval(() => {
      setStageIdx((i) => (i + 1) % STAGES.length);
    }, 1500);
    const elapsedTimer = setInterval(() => {
      setElapsedMs(performance.now() - startRef.current);
    }, 250);
    return () => {
      clearInterval(stageTimer);
      clearInterval(elapsedTimer);
    };
  }, []);

  const showElapsed = elapsedMs >= 5000;
  const elapsedSec = Math.floor(elapsedMs / 1000);

  return (
    <div className="inline-flex items-center gap-2.5 text-stone-700">
      <Sparkles size={16} strokeWidth={1.5} className="text-amber-500 animate-pulse shrink-0" />
      <span className="font-display italic text-[14px] sm:text-[15px]">
        {STAGES[stageIdx]}
      </span>
      {showElapsed && (
        <span className="text-[11px] tabular-nums text-stone-400 ml-1">
          {elapsedSec}s
        </span>
      )}
    </div>
  );
}
```

NOTE: `useState`, `useEffect`, `useRef` and `Sparkles` are already imported in this file (they're used widely). Confirm by grep if uncertain.

// True when the last message is a user message and we're not actively
