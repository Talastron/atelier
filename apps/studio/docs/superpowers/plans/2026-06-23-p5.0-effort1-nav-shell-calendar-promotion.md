# P5.0 Effort 1 — Nav shell + Calendar promotion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote Calendar to a top-level navigation pillar (3 clicks → 1), slim Lookbook, and make the Concierge the mobile centre element — the core fix for the buried-navigation problem, with no behaviour change to the views themselves.

**Architecture:** Re-wire the top-level navigation in `src/App.jsx`: edit the desktop sidebar nav list and the mobile bottom bar, add a `calendar` route to the render switch (rendering the existing `DiaryView`, which already contains the journal + calendar), remove the now-redundant `Diary` tab from Lookbook, and repoint the Insights→Diary hand-off. A final task extracts the nav markup into `src/nav/` modules (incremental modularization). No new views are built (the Today home is Effort 2; Calendar unification + Trips fold-in are Effort 3).

**Tech Stack:** React 18, Vite 6, Tailwind 4, lucide-react icons. Single-file `src/App.jsx` (~21k lines).

**Verification note (read first):** This project has **no automated test harness** (`package.json` has no `test` script). Per how the calendar feature shipped, every task is verified by: (1) `npm run build` passing, (2) an **import-audit** grep — any identifier newly referenced must be imported (this repo's white-screen bug class comes from un-imported identifiers), and (3) a live visual check after deploy. There are no unit-test steps because there is no framework to run them.

**Scope — explicitly deferred:**
- **Today home** → Effort 2 (the Daily Brief block is coupled inside `WardrobeView`; extracting it is its own effort). Default landing stays `wardrobe` for now.
- **Calendar unification** (past-worn on the grid, Month/Feed lens) and **Trips fold-in** → Effort 3. In Effort 1 the `calendar` route renders the *existing* `DiaryView` (its Journal/Calendar sub-toggle remains) and `Trips` stays a tab inside Lookbook.
