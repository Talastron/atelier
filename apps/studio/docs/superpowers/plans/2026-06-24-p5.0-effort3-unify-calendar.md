# P5.0 Effort 3 — Unify the Calendar surface Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Calendar pillar one coherent time surface — reconcile the "Calendar"/"The Diary" naming, relabel the lens to Month/Journal, and move **Trips out of Lookbook onto the Calendar** (an "Upcoming trips" shortcut + the existing per-day trip eyebrow), so planning lives where time lives.

**Architecture:** All changes are **in place** in `src/App.jsx`. Trip ownership (the `trips` useMemo, `openTripId`, the plan-a-trip hand-off, and the `TripsListView`/`TripDetailView` renders) moves from `OutfitBuilder` (lookbook mode) into `DiaryView` (the Calendar route). The Lookbook loses its Trips tab. `DiaryView` gains an `onDeleteTrip` prop threaded from `DigitalWardrobe`. No files are extracted (see "Explicitly deferred").

**Tech Stack:** React 18, Vite 6, Tailwind 4, lucide-react.

**Verification note:** No automated test harness. Each task: `npm run build` passes, import-audit (no un-imported identifiers — the white-screen guard), and live check after deploy.

**Explicitly deferred (NOT in this plan):**
- **Extracting the Calendar into `src/views/Calendar.jsx`.** It needs a prerequisite refactor — moving ~12 shared helpers (`itemImages`, `resolveOutfitItems`, `downloadIcs`, `weatherLabel`, `itemWearHistory`, `todayISO`, …) into `src/lib/` to avoid an App.jsx↔Calendar circular import. That's its own dedicated effort ("monolith dent"); doing it here would balloon scope and risk.
- **Full trip range-banding on the grid** (a brass band spanning a trip's days). The per-day trip eyebrow already exists (WearCalendar ~13521); richer banding is a later refinement.

- [ ] **Step 2: Derive trips + add openTripId/plan-trip state inside DiaryView**

Right after the existing `const [tab, setTab] = useState(…)` (~14918), add:
```jsx
const [openTripId, setOpenTripId] = useState(null);
// "Plan a trip" hand-off: switch to the Month lens and tell WearCalendar to
// open range-select mode. (Was handlePlanTripFromTrips in OutfitBuilder.)
const [planTripNonce, setPlanTripNonce] = useState(0);
const handlePlanTrip = () => { setTab('calendar'); setPlanTripNonce((n) => n + 1); };

// Trips are derived from scheduled days sharing a trip.id (was in OutfitBuilder).
const trips = useMemo(() => {
  const map = new Map();
  for (const [dateISO, entry] of Object.entries(schedules || {})) {
    if (!entry || !entry.trip || !entry.trip.id) continue;
    if (!map.has(entry.trip.id)) {
      map.set(entry.trip.id, { id: entry.trip.id, name: entry.trip.name || 'Trip', startISO: entry.trip.startISO, endISO: entry.trip.endISO, location: entry.trip.location || null, days: [] });
    }
    map.get(entry.trip.id).days.push({ dateISO, outfitId: entry.outfitId });
  }
  const today = todayISO();
  return [...map.values()].map((t) => {
    t.days.sort((a, b) => a.dateISO.localeCompare(b.dateISO));
    let status = 'upcoming';
    if (t.startISO <= today && t.endISO >= today) status = 'active';
    else if (t.endISO < today) status = 'past';
    return { ...t, status };
  }).sort((a, b) => {
    const order = { active: 0, upcoming: 1, past: 2 };
    if (order[a.status] !== order[b.status]) return order[a.status] - order[b.status];
    if (a.status === 'past') return b.startISO.localeCompare(a.startISO);
    return a.startISO.localeCompare(b.startISO);
  });
}, [schedules]);
```
(`useMemo`, `useState` are imported at line 1; `todayISO` is module-scope.)
