# Extract Views into `src/views/` Implementation Plan (p7.0)

> Follows p6.0 (pure helpers → `src/lib/`). Verification per task: `npm run build` (esbuild import resolution) + grep import-audit (moved-but-not-imported = white-screen) + batched live deploy. Move components verbatim; no behaviour change.

**Goal:** Lift the large React view components out of `App.jsx` into `src/views/`, after first extracting the shared UI primitives they depend on into `src/ui/` (so views import primitives from a neutral module, not from `App.jsx` — no cycle).

**Order (low-coupling first, from the scoping pass):**

### Batch 1 — shared UI primitives (prerequisite) + proof view
- `src/ui/EditorialHeader.jsx` — `EditorialHeader` (pure JSX; used by most views).
- `src/ui/toast.jsx` — `ToastContext`, `useToast`, `ToastProvider`.
- `src/ui/hooks.js` — `useEscapeKey`, `useCountUp`.
- `src/views/ShoppingDirectory.jsx` — smallest view (~115 lines); also move `SHOP_SEEDS`. Proves the `src/views/` pattern.

### Batch 2 — low-coupling views
- `src/views/InspirationView.jsx` (+ `AddInspirationModal`).
- `src/views/TodayView.jsx` (sibling cards `DailyBriefCard`/`WeekStrip`/`ConciergePrompt`/`DailyDigest` stay in App.jsx or co-move later).

### Batch 3 — medium-coupling
- `src/views/Calendar.jsx` — `DiaryView` + `WearCalendar` as a pair (both use `useToast` now in `src/ui/`).
- `src/views/WardrobeView.jsx` (+ `WardrobeCardImage`, `WardrobeFiltersSheet`, `WARDROBE_SORT_KEY`).

### Batch 4 — larger
- `src/views/ProfileView.jsx`, `src/views/FinanceView.jsx` (+ its chart sub-components).

### Batch 5 — the monster
- `src/views/OutfitBuilder.jsx` (~1.7K lines) + the outfit-slot model + its modal ecosystem. Last; biggest single shrink.

**Key dependency notes:**
- `EditorialHeader` is referenced by Inspiration/Finance/Profile/ShoppingDirectory — extract it first so each view imports `../ui/EditorialHeader.jsx`.
- `useToast` is needed by OutfitBuilder, WearCalendar, ProfileView — `src/ui/toast.jsx` unblocks them.
- `ToastProvider` still wraps the app in `App.jsx` (imported from `src/ui/toast.jsx`).
- `haptic` (small vibrate util) stays in App.jsx for now — not a view blocker.
