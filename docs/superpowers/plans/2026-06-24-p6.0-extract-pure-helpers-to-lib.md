# Extract Pure Helpers into `src/lib/` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Lift the ~3,150 lines of pure logic/data helpers at the top of `src/App.jsx` into focused, acyclic `src/lib/` modules, so future view extractions (Calendar, Wardrobe, Studio → `src/views/`) can import shared helpers from neutral modules instead of from `App.jsx` — eliminating the circular-import risk that has so far blocked splitting the 21K-line monolith.

**Architecture:** Pure move-refactor. Each task cuts a cohesive set of declarations out of `App.jsx` (verbatim, **no behaviour change**) into a new `src/lib/<module>.js`, adds `export` to each, and replaces them in `App.jsx` with a single `import { ... } from './lib/<module>.js'`. Modules form a strict DAG (no cycles): `taxonomy` ← `color` ← `items` ← `canvas`/`weather`/`ai`; `net` ← `canvas`; `receipts` and `ai` are leaf consumers.

**Tech Stack:** React 18, Vite 6 (esbuild), Firebase 11, lucide-react. **No test harness, no ESLint.**

---

## Verification Protocol (applies to EVERY task)

This codebase has no tests and no linter. Each task is verified by three layers, in order:

1. **Build** — `npm run build`. Esbuild **errors hard** if `App.jsx` imports a name the new module does not `export` (`"No matching export for import X"`). This is your guarantee the export list and import list agree. Expected: `✓ built in N.NNs`, no errors.

2. **Import-audit grep** — the white-screen guard. The build does **not** flag a moved identifier that is still *used* in `App.jsx` but no longer *imported* (esbuild leaves it as an undefined global → runtime `ReferenceError` → white screen). So for every task run, after the move:
   ```bash
   # (a) No moved identifier still has a local definition in App.jsx
   #     (i.e. you actually cut it, didn't leave a duplicate):
   grep -nE "^(export )?(const|function|async function) (NAME1|NAME2|...) " src/App.jsx
   #     Expected: NO matches.
   #
   # (b) Every moved identifier that App.jsx still references is in the new import line:
   #     visually confirm the `import { ... } from './lib/<module>.js'` line lists
   #     every NAME that grep shows is still used below it.
   grep -nE "\b(NAME1|NAME2|...)\b" src/App.jsx | head
   ```

3. **Live deploy smoke test** — batched, not per-task. After a batch of tasks builds green and passes the grep-audit, `npm run deploy` and confirm the app loads + the touched surfaces work on edit.myatelier.style. (No dev server — authDomain + App Check are bound to the prod domain.)

**Commit after each task** (green build + clean grep). **Deploy after each batch** (Tasks 1–4 = batch A "core"; Tasks 5–8 = batch B "subsystems").

**Move rule:** functions and constants move **verbatim**. Do not "improve" them mid-move — a refactor that also edits logic is two changes you can't bisect. JSDoc/comments travel with their function.

## File Structure

| New module | Responsibility | Imports |
|---|---|---|
| `src/lib/taxonomy.js` | Pure wardrobe vocabulary: category/style/material/care/colour constant lists & maps | (none) |
| `src/lib/color.js` | Colour science: rgb↔hsl, classify, hex lookup, palette derivation | `taxonomy` |
| `src/lib/items.js` | Pure item/outfit field accessors & derivations (wear stats, care, fit, price, ids, dates) | `taxonomy` |

| `src/lib/weather.js` | Weather fetch + labelling + appropriateness scoring + today's pick | `items` |
| `src/lib/net.js` | CORS-proxy chain, host backoff, product-URL scraping, brand search | `canvas` |
| `src/lib/receipts.js` | Receipt OCR text parsing | (none; RECEIPT regexes move here) |
| `src/lib/ai.js` | All Gemini calls (`*WithGemini`, concierge, manifesto, travel capsule) | `items`, `weather`, `taxonomy`, `../firebase.js` |

**Deferred (NOT in this plan):** React hooks/components still living at the top of `App.jsx` (`useEscapeKey`, `useToast`/`ToastProvider`, `useCountUp`, `EditorialHeader`, `ShareLookModal`) and the Firestore `user*Ref`/`*Doc` helpers. They are not the circular-import blocker for view extraction; extract them in a later effort once `src/lib/` is established.
